# 学习资源与参考（nano-vllm 与推理工程）

> 面向希望 **系统掌握大模型推理** 的学习者：先建立全局，再按模块深入，最后通过论文与官方文档补齐理论边界。

---

## 一、nano-vllm 推荐学习路线（顺序）

1. **入口与配置**  
   阅读 `example.py`、`nanovllm/config.py`，理解 `gpu_memory_utilization`、`max_num_batched_tokens`、`max_num_seqs`、`tensor_parallel_size` 等参数对行为的影响。

2. **引擎循环**  
   `LLMEngine`：`add_request` → `step` → `scheduler.schedule` → `model_runner.run` → `postprocess`。建立「一次 step 在干什么」的心智模型。

3. **序列与调度**  
   `Sequence`（长度、`block_table`、`num_cached_tokens`）与 `Scheduler`（prefill 优先、decode、`preempt`）。对照课程文档中的时序图更佳。

4. **KV Cache 与 Block**  
   `BlockManager`（xxhash 链式哈希、`allocate`/`deallocate`/`may_append`）与 `ModelRunner.allocate_kv_cache`（张量形状与块数计算）。

5. **注意力与 FlashAttention**  
   `layers/attention.py`：`store_kvcache` + `flash_attn_varlen_func` / `flash_attn_with_kvcache`，结合 `utils/context.py` 中的 `cu_seqlens_*`、`slot_mapping`、`block_tables`。

6. **张量并行**  
   `layers/linear.py`、`layers/embed_head.py`、`models/qwen3.py`：列并行、行并行、QKV 合并、LM Head 的 `gather`。

7. **CUDA Graph**  
   `ModelRunner.capture_cudagraph` 与 `run_model` 中 prefill / decode 分支，理解 `graph_bs` 与 `enforce_eager`。

8. **RoPE 与模型块**  
   `rotary_embedding.py`、`Qwen3DecoderLayer` 中的 Pre-Norm + 残差、`SwiGLU`（`SiluAndMul`）。

9. **采样与工程细节**  
   `Sampler`（Gumbel-max 技巧）、`SamplingParams`（温度下界）、`loader.py`（safetensors + `packed_modules_mapping`）。

建议 **每阶段配合本地单步调试或打印 tensor 形状**，比只读不写更快建立直觉。

## 三、前置知识学习资源

### 1. PyTorch

- 官方文档：[https://pytorch.org/docs/stable/index.html](https://pytorch.org/docs/stable/index.html)  
- 重点：`torch.nn`、`autograd` 基础、`Distributed` 概念、与 CUDA tensor 的内存与 stream（为理解 Graph 与异步拷贝打基础）。

### 2. CUDA

- NVIDIA CUDA C++ Programming Guide  
- 重点：grid/block/thread、共享内存、合并访问；不必一上来写大 kernel，能读懂 **Triton 与 PyTorch 扩展** 即可与 nano-vllm 对齐。

### 3. Transformer

- 经典博客与视频（Attention Is All You Need 图解）  
- 重点：自注意力复杂度、KV Cache 在解码阶段的含义、RoPE 与位置信息的关系。

## 七、学习节奏建议

- **第 1 周**：跑通示例，跟完 `LLMEngine` + `Scheduler` 主路径。  
- **第 2 周**：KV + Block + Attention，能口述 prefill/decode 数据准备差异。  
- **第 3 周**：TP + RoPE + Sampler，能对照 `qwen3.py` 与 `linear.py` 画图。  
- **持续**：读 vLLM 对应模块差异（前缀缓存、更完整调度），形成对比笔记。

坚持 **「能讲 + 能指源码 + 能画一张图」**，面试会轻松很多。
