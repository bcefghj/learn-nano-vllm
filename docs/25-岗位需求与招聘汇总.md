## 二、分岗位技能要求分析

### 1. 推理框架与系统（vLLM / SGLang / 自研）

- **共性**：理解 **连续批处理、KV Cache、PagedAttention、调度与显存池**，能读源码、做二次开发或插件。
- **nano-vllm 对齐点**：通读 `Scheduler`、`BlockManager`、`ModelRunner`、`Attention` 四层协作，能画出 **从请求进入到 step 输出** 的数据流。

### 2. CUDA / 算子 / 内核

- **共性**：kernel 融合、内存合并访问、与 PyTorch/Triton 的衔接；部分岗位强调 **TensorRT** 或 **CUTLASS**。
- **nano-vllm 对齐点**：`layers/attention.py` 中 **Triton `store_kvcache_kernel`**；理解 FlashAttention 接口与 `slot_mapping` 的配合。

### 3. C++ 与线上工程

- **共性**：高性能服务、多线程/多进程、与 GPU 驱动和推理 Runtime 的集成。
- **nano-vllm 对齐点**：Python 侧可补充 **多进程 TP**（`ModelRunner` + `SharedMemory` + `Event`）的设计理解，并说明若迁移到 C++ 服务层的接口划分思路。

### 4. 性能指标与方法论

- **共性**：**吞吐（tokens/s）**、**首 token 延迟（TTFT）**、**每步 decode 延迟**、显存占用与 OOM 边界。
- **nano-vllm 对齐点**：`LLMEngine.generate` 中对 prefill/decode throughput 的统计方式；`gpu_memory_utilization` 与 `num_kvcache_blocks` 的关系。

## 三、如何用 nano-vllm 项目经验匹配上述岗位

建议在简历与面试中按 **「问题 → 方案 → 代码落点 → 指标」** 表述，例如：

1. **KV Cache 与显存**：说明 `allocate_kv_cache` 如何用峰值显存、`block_bytes` 推导 block 数量；张量形状 `[2, L, B, block_size, kv_heads, head_dim]` 与层绑定方式。
2. **PagedAttention 与前缀缓存**：说明 `BlockManager.compute_hash` 的链式前缀哈希、`allocate` 中 cache hit 与 `ref_count` 的配合。
3. **连续批处理与抢占**：说明 `schedule` 中 **prefill 优先**、decode 阶段 `can_append` 失败时的 `preempt` 与 `deallocate`。
4. **FlashAttention 双路径**：prefill 用 `flash_attn_varlen_func`，decode 用 `flash_attn_with_kvcache`，以及 `cu_seqlens_*` 与 `block_table` 的含义。
5. **张量并行**：`ColumnParallelLinear` / `RowParallelLinear` / `QKVParallelLinear` 分片与 `all_reduce`、`ParallelLMHead` 的 `gather`。
6. **CUDA Graph**：仅 decode、离散 `graph_bs`、与 `enforce_eager` 的取舍。

这样可以把「读过 toy 项目」升级为 **「能对接生产级推理系统模块」** 的叙事。

## 四、校招 vs 社招差异（推理方向）

| 维度 | 校招常见侧重 | 社招常见侧重 |
|------|----------------|----------------|
| 基础 | 数据结构、操作系统、网络、机器学习基础 | 过往项目深度、线上问题排查 |
| 系统 | 能讲清原理 + 简单手撕 | 端到端优化案例、资源与 SLA |
| 工程 | 代码规范、学习能力、实习 | 稳定性、协作、交付节奏 |
| 薪资 | 按档定薪 + sp/ssp | 与年限、职级、团队强相关 |

校招同学可强调：**源码阅读 + 小规模实验（bench）+ 清晰图解**；社招同学需准备：**瓶颈定位方法、数据、复盘**。

## 五、薪资范围分析（仅供参考）

- **一线互联网 / 大模型业务**：应届总包跨度大，与学历、实习、面试表现强相关；推理方向若与 **GPU、内核、引擎** 强绑定，往往属于 **研发薪资带宽中上段**。
- **专家 / 架构**：月薪区间常见于 **数万至十余万**，通常含股票/期权与绩效，表格中的「5–7 万/月」等为 **坊间区间**，以 offer 为准。
- **地域**：北上深杭通常高于其他城市；同一公司不同 base 也有差异。

**建议**：把「薪资」作为信息检索与谈判参考，面试准备仍以 **技术深度与项目匹配度** 为主。

---

## 六、小结

推理优化岗位 **高度交叉**（系统 + 模型 + GPU），nano-vllm 适合作为 **最小闭环**：从 `LLMEngine` 到 `Attention` 全链路可调试、代码量可控。求职时把 **调度、显存、注意力、并行、图捕获** 五条主线讲透，并各对应 1–2 处源码行号或函数名，会显著强于泛泛的「了解 vLLM」表述。
